# 安全说明

## 安全边界

首版面向本地单用户或受控可信环境，不是默认可公开访问的多租户服务。主要高风险资产包括 Bilibili Cookie、Cookie 加密主密钥、账号脱敏信息、签名媒体地址、下载产物、转写和分析结果。

系统不提供绕过付费、DRM、验证码或平台风控的能力，也不提供 Cookie 分享或账号凭据传播能力。使用者必须确保自己有权访问、下载和处理目标内容。

## Cookie 生命周期

1. 前端只通过受限 JSON 文件上传，不写入浏览器持久存储。
2. 服务端限制文件大小、条目数量、域、path、secure 与过期语义。
3. 原始上传内容解析后不作为文件保留，API 和前端状态不返回 Cookie 值。
4. 仅会话模式只在进程内使用；本机记住模式使用 Fernet 认证加密后存储。
5. 清除登录态时删除内存状态、数据库密文和相关身份缓存，不连带删除用户媒体产物。

Docker 的加密主密钥位于 `bili-insight-secrets` 卷，数据库位于 `bili-insight-runtime` 卷。镜像、Git 仓库和 `.env` 都不包含密钥。密钥卷丢失后不能恢复已加密 Cookie，应在界面清除旧登录态并重新上传 Cookie。

## 网络安全

- 默认网关只绑定 `127.0.0.1`，后端不发布主机端口。
- Bilibili 链接使用协议、域名和地址白名单校验，跳转目标需要逐跳重新校验。
- Cookie 依照 CookieJar 域和 path 规则发送，并限制到明确允许的 Bilibili 服务域。
- 上游请求具有连接/总超时、最大响应体、并发限制和有限退避重试。
- 页面标题、简介、字幕、OCR、弹幕和分析文本均按不可信内容处理。
- Nginx 设置 CSP、`frame-ancestors`、`nosniff`、Referrer Policy 与 Permissions Policy。

公网或局域网代理要求见 [部署与运维说明](DEPLOYMENT.md)。没有 HTTPS 和应用鉴权时，不得开放非回环监听。

## 文件与进程

- 文件名和产物路径由服务端净化和分配，API 不接受任意服务器绝对路径。
- FFmpeg/FFprobe 使用参数数组启动，不通过 shell 拼接用户输入。
- 临时文件与完整产物分离，校验成功后再原子发布。
- 容器以非 root 用户运行，删除 capabilities，并限制网关可写目录。
- 产物内容接口通过受控标识读取并支持 Range，不把存储路径作为客户端参数。

## 仓库与 CI

`.gitignore` 和 `.dockerignore` 排除真实 Cookie、环境文件、密钥、数据库、日志、上传缓存、临时目录、下载和分析产物。CI 的仓库安全任务还会拒绝凭据文件名、固定私钥和高置信度会话凭据形态。

CI 只使用脱敏固定响应与可合法分发的小型媒体样本，不携带个人 Cookie，也不依赖真实 Bilibili 登录接口。真实上游验证应由授权人员在隔离、限频的本地环境手动执行。

## 日志和诊断

日志过滤 Cookie、Authorization、CSRF、账号 ID、签名查询参数和服务器绝对路径。前端不得把 Cookie 写入控制台、状态持久化或错误监控。诊断导出仍应最小化共享范围和保留时间。

## 凭据疑似泄露时

1. 立即在 Bilibili 官方渠道撤销相关会话或退出所有设备。
2. 停止应用对外入口并清除应用登录态。
3. 检查 Git 历史、CI 日志、容器日志、备份和诊断包的暴露范围。
4. 若加密主密钥泄露，先清除所有持久 Cookie 密文，再轮换密钥并重新上传有效 Cookie。
5. 恢复服务前验证仓库安全检查、日志脱敏和网络访问控制均通过。
